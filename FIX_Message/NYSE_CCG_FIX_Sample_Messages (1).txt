1  Introduction
The following test cases are being provided to aid in your coding to NYSE CCG FIX 4.2. CCG FIX can 
vary slightly from standard FIX in several message types, namely Cancel Replace and Cancel Reduce 
tags 37 and 41. There are also several user tags in the 9000 series that contain information that 
could be valuable in your message and execution processing (eg. Tag 9579 ExpERCReferenceNumber — 
the newly expanded Activity ID). Please refer to the NYSE CCG FIX Specification document dated 
2/8/10, (available from http://www.nvse.com/content/publications/1043269645619.html) for further 
clarification. Should you have any questions regarding NYSE CCG FIX, please contact Member Firm 
Services at firmtestinq@nyx.com or 1-866-873-7422.


Please note:

In all of the following sample messages, the “Raw Message” displays the FIX tags in header, body, 
trailer format, whereas the tables display the FIX tags in numerical order.

Throughout this document, the use of ^A in the Raw Messages represents a delimiter.

FIX tags 17 and 19 are now available in the new expanded format in both UAT and production per 
customer request (reference: http://www.nvse.com/pdfs/NYSETECH INFOMEMO022609.pdf). Please contact 
Member Firm Services at firmtestinq@nyx.com or 1-866-873-7422 to have this feature enabled.

Drop Copy recipients will receive copies of orders, cancel requests, cancel replace requests, 
execution reports, UR OUTs and/or Replaced messages depending on the specific message types they 
have subscribed to. The messages contained in this document are exactly as they would appear on a 
drop copy session. CCG Drop Copy does not return order acknowledgements, pending cancels or pending 
replace messages.
